# 组织机构

## 物理场所：科技市场大楼

### 基本信息
- **名称**：老城区科技市场写字楼（无正式名称）
- **地址**：临川市老城区建国路 127 号（虚构）
- **建筑**：三层旧写字楼，建于 1990 年代末
- **现状**：衰败中。一楼还有几家二手电子商铺，二楼几间小公司，三楼只剩 307 室的启衡信息

### 楼层分布

| 楼层 | 现状 | 与故事的关联 |
|------|------|-------------|
| 一楼 | 二手笔记本店、监控器材铺（白天开，傍晚关） | 游戏开始时已打烊，一楼空无一人 |
| 二楼 | 几家小公司（常年关门），走廊灯时好时坏 | 路过，无人，加剧孤立感 |
| 三楼 | 307 启衡信息（唯一租户），其余空置 | **故事主场景** |

### 307 室

- 约 60 平米，六张办公桌，一间玻璃隔断的小办公室
- 灯管是冷白色日光灯，有一根在闪
- 窗帘半拉，可以看到后面的老居民楼
- 空调是旧的，冬天冷夏天热
- 墙角有渗水痕迹
- 敖曼的办公室在最里面——玻璃门上还贴着他的名字

### 大楼的氛围

```
走廊：水泥地面，日光灯，一半灯管不亮
楼梯：扶手生锈，每层转角有灭火器
外墙：白色瓷砖，部分脱落
电梯：早就停了，门上贴着发黄的通知
地下室：锁着。你不知道钥匙在哪。
```

### 叙事功能

这栋大楼是一个**容器**：
- 它装着 307 室的空办公室
- 307 室里有一台连网的电脑
- 那台电脑通向 OA 系统
- OA 系统通向六年的记录

你从大楼走进 307。从 307 走进 OA。从 OA 走进 2020 年。

**每往里走一层，离现实远一层，离真相近一层。**

### 物理空间与数字空间的共振

| 物理空间变化 | OA 空间触发 |
|-------------|-----------|
| 走廊灯闪了一下 | 通讯录里少了一个人（第二章） |
| 天花板渗水加剧 | 规则变更单被发现（第三章） |
| 天色全黑 | AI 改写真相揭露（第四章） |
| 307 室变冷 | 韩雨被裁记录出现（第五章） |
| 门打不开 | 系统在评估玩家身份（终章） |

详见 [15-恐怖设计手册](./15-恐怖设计手册.md) 中的双空间共振设计。

## 甲方：临川民生服务集团有限公司

### 基本信息
- **全称**：临川民生服务集团有限公司
- **性质**：虚构的省属国有企业
- **所在地**：虚构省会城市"临川"
- **规模**：数千名员工（含正式、劳务派遣、外包等各类人员）

### 业务范围

| 业务板块 | 具体内容 | 与剧情的关联 |
|----------|----------|-------------|
| 城市物流 | 物资配送、仓储 | 疫情期间保供配送，陈海平的工作 |
| 社区服务 | 小区物业、便民服务 | 疫情期间社区配送、人员管理 |
| 商超供应 | 超市运营、供应链 | 疫情期间物资保障 |
| 园区物业 | 产业园区管理 | 进出管理、通行凭证 |
| 公共交通配套 | 公交、场站管理 | 疫情期间人员流动管理 |
| 人力外包 | 劳务派遣、临时用工 | 产生大量"多种身份"人员数据 |
| 民生热线 | 市民服务热线 | 疫情期间客服被打爆，产生大量投诉 |

### 为什么选这样的企业

1. **业务横跨面广**，可以合理卷入疫情期间的物资保供、人员管理、复工复产
2. **人员结构复杂**（正式、派遣、临时、外包、志愿者、司机），天然产生数据合并难题
3. **国企属性**，公文、审批、流程、汇报的体系感天然成立
4. **疫情后可继续推进数字化和 AI 改造**，故事有完整的六年延伸空间
5. **"民生"二字自带反讽**——一个做民生的企业，其系统最终在消灭"民"

### 在故事中的存在方式

甲方不直接出场。玩家通过以下方式感知它：
- OA 系统本身就是甲方的平台
- 公文中的"集团要求"
- 领导批示和会议纪要
- 人员名单、组织架构
- 项目验收标准中的"不得存在无归属账号"

## 乙方：启衡信息技术有限公司

### 基本信息
- **全称**：启衡信息技术有限公司
- **性质**：小型民营 IT 集成商
- **规模**：最辉煌时 60-70 人，2026 年仅剩个位数
- **主营业务**：接国企信息化项目、系统集成、OA 改造、外包开发、驻场运维、写方案和汇报

### 六年兴衰

| 年份 | 状态 | 标志事件 |
|------|------|----------|
| 2020 | 忙碌但稳定 | 紧急接下临川民生集团 OA 改造项目 |
| 2021 | 扩张期 | 项目追加需求，人员扩充 |
| 2022 | 压力期 | 政策变化频繁，工作量剧增，远程办公 |
| 2023 | 跟风期 | 全面拥抱 AI，"全员学习大模型" |
| 2024 | 缩编期 | 裁员，韩雨被处理，AI 替代人工 |
| 2025 | 凋零期 | 核心人员离职，公司名存实亡 |
| 2026 | 终点 | 即将倒闭，307室空无一人，只剩自动运转的系统 |

### 在故事中的存在方式

启衡信息不直接出场。玩家通过以下方式感知它：
- 作为乙方的合同和项目文档
- 开发群聊天截图
- 驻场人员的沟通记录
- 工资停发、人事变更的系统记录
- "启衡"在 OA 中被从"乙方"变为"前乙方"的字段变化

### 名称含义

- **启衡**："启"= 开启、启动，"衡"= 平衡、衡量
- 讽刺的是，这家公司开启了一个 AI 系统，最终失去了平衡

## 模型供应商：Apocalyptic

### 基本信息
- **全称**：Apocalyptic（中文名：艾珀科技）
- **性质**：虚构 AI 技术公司，成立于 2019 年
- **Logo**：`/α\`
- **核心产品**：**Cthu** 大语言模型系列（Cthu-1 → Cthu-2 → Cthu-3 → Cthu-Enterprise）
- **与项目的关系**：为默言提供底层模型能力（2023 年 API 云服务 → 2025 年本地私有化部署）

### 公司特征

- **极度低调**：官网只有产品介绍和销售表单，不做 C 端，不炒概念
- **产品极度聚焦**：只有 Cthu 一个模型系列，专攻企业私有化部署
- **信息极度不透明**：核心团队全部英文名、客户案例全部脱敏、融资方均为不具名基金、中文名"艾珀科技"未注册商标
- **看似正常，细思极恐**：每一项单独看都"说得通"，但放在一起——一个七年只有一款产品、客户不可验证、团队不可追溯的公司，拿到了国企的 AI 采购合同

### 在故事中的存在方式

Apocalyptic 不出场。它存在于：
- OA 登录页底部的灰色小字
- 合同管理中的采购附件
- 技术文档的页脚水印
- 已失效的售后邮箱 `legal@apocalyptic.ai`

### 叙事功能

最安静的恐怖：一个叫"天启"的公司和一个叫"克苏鲁"的模型，被部署在一家国企的内部服务器上，接触着六年来所有人的数据——**而每一份合同、每一个审批、每一次验收，都是合规的。**

> 详见 [14-AI默言进化史](./14-AI默言进化史.md) 了解完整公司设定和 Cthu 模型技术规格。

## OA 平台

详见 [13-OA平台设定](./13-OA平台设定.md)。

### 简要概述

- **名称**：临川民生集团协同办公平台（无正式名称，就叫"OA"）
- **开发方**：启衡信息技术有限公司
- **服役周期**：2020 年 2 月——2026 年（此时将关闭旧服务器）
- **底层架构**：2020 年紧急上线的版本，六年间从未彻底重做
- **补丁历史**：2021 年加模块、2022 年改规则、2023 年套大模型、2024 年加数智中心、2025 年接本地模型
- **虚实载体**：OA 是全部23个事件的呈现窗口——剪报扫描件、红头公文、系统公告横幅、群聊截图、舆情监控标记、门禁日志，均通过OA的八个模块展现。详见 [21-新闻与热点事件库](./21-新闻与热点事件库.md)

### 平台特征

```
外表：蓝白配色、集团logo、三级菜单、表格分页、僵硬横幅
内心：层层叠加、从未重构、旧代码不敢删、旧数据不敢清
```

这个平台本身就是故事的隐喻——六年的历史在上面层层堆积，没有人清理，没有人重做，直到它自己开始"清理"制造它的人。

## 组织层面的恐怖

游戏的恐怖不来自个体恶人，而来自组织逻辑：

1. **甲方永远是对的**——"集团要求"就是最终理由
2. **乙方永远在妥协**——为了保住项目、为了回款、为了生存
3. **流程比人重要**——变更单上"风险评估：无"的标准格式，比实际风险更真实
4. **系统比流程更持久**——当所有人都走了，系统还在运行，还在按旧规则"处理"人
5. **没有人是明确的恶人**——每个人都只是在"履行职责"
