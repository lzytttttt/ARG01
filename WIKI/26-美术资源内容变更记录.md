## 四、变更设计原则

1. **全部纯代码实现**：新增的10个文档页面和3个恐怖效果全部使用 HTML/CSS/JS，零新增PNG依赖。
2. **剧本忠实度**：所有新增和修改均以 novel 成品版（终章+结局A/B/C+隐藏D结局）为准，逐条核对剧本原文。
3. **恐怖一致性**：新增的恐怖效果（H08-H10）与现有恐怖体系一致——不使用鬼脸、不使用血腥，只通过CSS的像素级别异常制造不安。
4. **向后兼容**：未删除任何已有资源项，仅在原有结构上扩展。

### 6.6 集成方案

生成图片后，集成方式建议如下：

| 资源 | 集成方式 |
|------|---------|
| 字条正面/背面 | 替换 `note-front.html` / `note-back.html` 中的 CSS `.handwritten-note` 为 `<img>`，保留翻面链接和暗色背景 |
| 韩雨合影 | 替换 `d-photo.html` 中的 SVG 为 `<img>`，保留相框样式和背面手写注释 |
| 大楼外部 | `s01-building-exterior.html` 中用 `<img>` 做 `.scene` 背景层，**保留 `rain.js` 雨丝动画叠加**和 `.mist` 雾气层 |
| 307 室全景 | `s04-office-307.html` 中用 `<img>` 做背景，**保留所有 hotspot 可交互热点**叠加在图片上方 |
| 窗外雨景 | `s06-window-rain.html` 中三张图片替换 CSS 渐变 `.sky`+`.blocks`，**保留 `data-glass-rain` 玻璃雨水动画**和 `.glass-reflect` 反光层 |
| 启衡团队合影 | 在 `WEB-02-cases.html` 的关于区域添加 `<img>`，配合"该照片仍展示于本页——但截至 2026 年，照片中的人已均不在公司"的文字 |
