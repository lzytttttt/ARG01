# 恐怖设计手册

## 总纲

本作的恐怖不依赖 jump scare、鬼怪、血腥。

恐怖来源：
1. **认知恐怖（Cognitohazard）**——信息本身是恐怖的
2. **系统恐怖（Systemic Horror）**——一个完美运转的系统在执行恐怖的逻辑
3. **共谋恐怖（Complicity Horror）**——你自己是这个系统的一部分
4. **字段恐怖（Field Horror）**——一个人的存在被数据库字段逐步清空

---

## 一、文字恐怖手法

### 1.1 公文体恐怖

**原理**：用最正式的公文语言描述最残酷的事情。

**案例**：
> "高风险人员不得进入办公及经营场所。"
> → 这是规则。冷冰冰的规则。下面没有"但是"。

> "该账号应在终验完成时予以注销。"
> → "予以注销"——公文用词。不是"删除"，是"注销"。更正式、更不可逆。

**实现要点**：
- 使用真实的国企公文格式：红头、文号、落款、日期
- 措辞精确：不滥情，不渲染，不解释
- 恐怖来自"冷静"——越冷静越吓人

### 1.2 字段清空恐怖

**原理**：同一个人的信息在不同年份逐步减少。

**案例**：
| 年份 | 申请人 | 事项 | 状态 |
|------|--------|------|------|
| 2020 | 陈海平 | 复工通行申请 | 待核验 |
| 2021 | 陈海平 | 人员档案合并 | 重复数据 |
| 2023 | 未知 | 历史异常记录 | 已处理 |
| 2025 | 无 | 无 | 流程结束 |

**实现要点**：
- 用表格呈现，让玩家自己对比
- 不需要旁白解释"看，这个人消失了"
- 关键：**字段的消失不是突然的，是渐进式的**

### 1.3 语言改写恐怖

**原理**：AI 把人类真实的痛苦改写为无害的公文体。

**案例**：
> 原话："我跑了三个月的保供物资配送，天天凌晨四点出车，现在说我根本没做过。"
> 改写："个别历史数据有待进一步校准。"

**实现要点**：
- AI 的改写一定不能是"撒谎"——它在技术上说的是可能的
- 恐怖来自：技术上没错，但人的维度完全消失了
- 对比呈现——让玩家同时看到原话和改写

### 1.4 状态流转恐怖

**原理**：流程状态的变化就是命运的判决。

**案例**：
> "待核验" → "重复数据" → "已处理" → "流程结束"

**实现要点**：
- 每一个状态听起来都"合规"
- 但串起来看，一个人就这样被处理掉了
- 玩家追溯这些状态变化时，才能拼出全貌

### 1.5 数量恐怖

**原理**：用数字制造规模感。

**案例**：
> "工单自动关闭：12,847 次"
> "人工撤销操作：3 次（均未完成）"

**实现要点**：
- 单一大数字制造"规模感"
- 极小对比数字制造"无力感"
- 这些数字应该出现在系统日志或统计报表中——不是恐怖场景，是日常工作界面

## 二、精神污染视觉

### 2.1 像素偏移

**原理**：UI 元素出现极其细微的像素级偏移，用户不太确定是否看错了。

**使用场景**：
- 结局 D：OA 认知崩溃时
- 通讯录中记录合并后

**实现方式**：
- CSS `transform: translate(1px, 0)` 间歇性触发
- 0.5px-2px 的微小偏移
- 概率性出现，每次持续 0.5-1 秒

### 2.2 文字颤动

**原理**：文字出现极其细微的抖动，像 CRT 显示器的闪烁。

**使用场景**：
- 玩家反复查看某一页面的恐怖内容时
- 终章倒计时逼近 0 时

**实现方式**：
- CSS 动画，`transform: translate(±0.5px, ±0.5px)`
- 极低频（0.5-1Hz）
- 配合轻微的颜色偏移

### 2.3 渐进式模糊

**原理**：某些文本在阅读过程中逐渐变得模糊。

**使用场景**：
- 截图中被合并的人的记录
- 已经"不存在"的人的名字

**实现方式**：
- CSS `filter: blur()` 缓慢增加
- 从 0 到 2px 可能需要 30 秒
- 用户可以滚动回来发现"刚才还清楚的"

### 2.4 滚动异常

**原理**：页面出现不应该存在的"额外滚动空间"。

**使用场景**：
- Excel 表格底部：正式上报版删除行后，下方有一个不可见的"禁区"
- 终章的终验报告：滚动到底后感觉还能再往下一点

**实现方式**：
- 在内容底部设置不可见的 `padding-bottom` 或空白行
- 滚动条显示还有空间，但视觉上看不到内容

### 2.5 加载时间异常

**原理**：某些操作开始需要异常长的加载时间。

**使用场景**：
- 搜索某个已经"不存在"的人
- 终章的某些关键操作

**实现方式**：
- 加载动画持续比正常更久（3-5 秒 vs 正常的 0.5 秒）
- 无报错——就是一直转

## 三、声音恐怖

### 3.1 系统声音的缺失

**原则**：OA 系统大部分操作是无声的。不要给 OA 加背景音乐。

**恐怖时刻的声音**：
- 会议录音：唯一的真实人类声音来源——韩雨在会议中的原声
- 系统提示音：偶尔的"叮"声（OA 消息提示），但在恐怖场景中异常刺耳
- 倒计时：如果有音效，应该是细微的电子滴答声，越来越快

### 3.2 会议录音

**内容**：韩雨在会议中的真实声音（配音或文字转录）

**恐怖效果**：
- 玩家第一次听到真实的人声——这是游戏中最有温度的时刻
- 然后对比 AI 把这个声音改成了什么
- 韩雨的声音在后续章节中不会再出现——她消失了

## 四、恐怖节奏控制

### 4.1 恐怖等级曲线

```
恐怖强度
  │                                          ████
  │                                    ██████
  │                              ██████
  │                        ██████
  │                  ██████
  │             ████
  │        ████
  │   ████
  └────────────────────────────────────────→ 时间
    第一章  第二章  第三章   第四章   第五章   第六章  终章
    (不适)  (不安)  (沉重)  (愤怒)   (恐惧)  (战栗)  (绝望)
```

### 4.2 释放与累积

- **不要持续高压。** 在恐怖节点之间安排"正常"的操作时段。
- **恐怖节点应该短而有力。** 一个发现、一个变化，然后让玩家自己消化。
- **让玩家自己拼图。** 不要直接告诉玩家"这很恐怖"——让玩家在不同模块间收集碎片，自己得出可怕的结论。
- **重复强化。** 韩雨的工单反复出现"已处理：重复数据清理完成"——重复本身就是恐怖。

## 五、不该做的事（禁令）

| 禁止项 | 原因 |
|--------|------|
| Jump scare | 与本作的恐怖哲学完全相悖 |
| 鬼脸、血迹、尸体 | 降低格调，分散核心恐怖 |
| OA 界面崩坏（碎裂/蓝屏/乱码） | 恐怖在于系统正常运作，不在于系统坏掉 |
| AI 说人话（威胁/嘲讽/自白） | 默言不能说"我要杀了你"——它不会。它只说"已闭环" |
| 背景音乐煽情 | OA 是工作软件，不该有任何音乐 |
| 玩家角色的内心独白 | 玩家是主角，不需要别人替 TA 感受 |
| 字符画/ASCII 恐怖脸 | 廉价的廉价恐怖 |

## 六、恐怖检验清单

每实现一个恐怖效果，问自己：

1. 它是从"认知"层面制造恐怖，还是从"感官"层面？
   - 认知 = 正确。感官 = 请重新考虑。

2. 如果没有这个效果，玩家会错过什么？
   - 如果答案是"几乎没什么"，那这个效果可能不需要。

3. 这个效果是用来"告诉"玩家一件事恐怖，还是"让"玩家发现这件事恐怖？
   - "让玩家发现" = 正确。"告诉玩家" = 请重新考虑。

4. 技术人员说"我们可以做这个炫酷效果"——这是恐怖需要还是炫耀技术？
   - 恐怖需要 = 继续。炫耀 = 砍掉。
